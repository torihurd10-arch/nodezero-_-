How Networks Work

When you open a website, data travels through a series of systems before it reaches your screen. Here's how it works:

🌐 IP Address

Every device on a network has an IP address — its unique location identifier. Like a street address for computers. Example: 192.168.1.1

🔤 DNS

Domain Name System translates human-readable names (google.com) into IP addresses (142.250.80.46). Like a phone book for the internet.

🔒 HTTP / HTTPS

HTTP is the protocol browsers use to request web pages. HTTPS is the encrypted version — the padlock in your browser. Always prefer HTTPS.

🚪 Ports

Ports are communication channels on a device. HTTP uses port 80, HTTPS uses 443, SSH uses 22. Think of them as doors into a building.

📦 What Happens When You Visit google.com?

  1. Your browser asks DNS: "What IP is google.com?"
  2. DNS replies: "142.250.80.46"
  3. Your browser sends an HTTP request to that IP on port 443
  4. Google's server sends back the web page
  5. Your browser renders it on screen
