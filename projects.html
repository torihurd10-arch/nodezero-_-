PROJECT 1 — Hardware

System Specs Audit

+50 XP

~15 min

Document your computer's hardware specs. This is the first thing IT professionals do when troubleshooting a machine.

▸ Show Instructions

Steps:

  1. Press Win + R, type msinfo32, press Enter (Windows) OR run system_profiler SPHardwareDataType in Terminal (macOS)
  2. Record: CPU model, CPU speed (GHz), Number of cores
  3. Record: Total RAM installed
  4. Open File Explorer → right-click C: → Properties → note disk type and free space
  5. In Task Manager (Ctrl+Shift+Esc) → Performance tab → observe live CPU, RAM, Disk usage
  6. Write everything in the answer box below

Verification Checklist:

  • ✓ I know my CPU model and speed
  • ✓ I know my total RAM amount
  • ✓ I know if my drive is HDD or SSD
  • ✓ I can open Task Manager and read performance data

Job Connection:

Help-desk technicians document system specs before every support ticket. This exact skill is tested in CompTIA A+ certification (exam objective 1.1).

PROJECT 2 — Networking

Network Diagnostic Lab

+60 XP

~20 min

Run real network diagnostic commands used by IT professionals every day. Record your findings.

▸ Show Instructions

Commands to run (open Command Prompt or Terminal):

  1. ipconfig (Windows) or ifconfig (macOS/Linux) — find your local IP address
  2. ping google.com — test internet connectivity. Record response time (ms)
  3. ping 8.8.8.8 — test IP connectivity. If this works but ping google.com fails: DNS problem!
  4. nslookup google.com — see which IP DNS resolves google.com to
  5. ipconfig /flushdns (Windows) — clear DNS cache
  6. Record all results and explain what each tells you

Verification Checklist:

  • ✓ I know my computer's local IP address
  • ✓ I can test internet connectivity with ping
  • ✓ I understand the difference between IP and DNS failure
  • ✓ I know how to flush DNS cache

Job Connection:

These 5 commands cover 80% of first-level network troubleshooting. "Run ping and ipconfig, then call us back" is heard on help desks worldwide.

PROJECT 3 — CLI

Terminal Navigation Challenge

+70 XP

~20 min

Complete 8 CLI tasks using only the terminal. No mouse. No GUI. This is what IT pros do on servers without desktops.

▸ Show Instructions

Complete all 8 tasks in order:

  1. Open a terminal. Run pwd — write down your current directory
  2. Navigate to your Desktop: cd ~/Desktop (macOS/Linux) or cd %userprofile%\Desktop (Windows)
  3. Create a new folder: mkdir nodezero-lab
  4. Navigate into it: cd nodezero-lab
  5. Create a text file: echo "I completed Level 0 CLI lab" > notes.txt
  6. List the files: ls or dir — verify notes.txt exists
  7. Show file contents: cat notes.txt (macOS/Linux) or type notes.txt (Windows)
  8. Go back up: cd .. — verify you're back on Desktop

Verification Checklist:

  • ✓ I can navigate directories using cd
  • ✓ I can create folders with mkdir
  • ✓ I can create files with echo/redirect
  • ✓ I can list and read file contents
  • ✓ I completed all 8 steps without using a mouse

Job Connection:

Linux servers often have no GUI. Every command you just ran is used daily by sysadmins and DevOps engineers managing remote servers.

PROJECT 4 — Security

Personal Security Audit

+80 XP

~25 min

Audit your own security posture using the same checklist security professionals use for enterprise systems — applied to your personal accounts.

▸ Show Instructions

Complete this security checklist:

  1. Check if you reuse passwords: Do any 2 of your accounts share the same password? (Yes/No)
  2. Check your email account at haveibeenpwned.com — has it been in a data breach?
  3. Enable 2FA on one account you don't already have it on (email, GitHub, etc.)
  4. Identify one phishing red flag: Go to your spam folder — find an email with a suspicious sender domain
  5. Check your Windows Defender / macOS Gatekeeper is enabled: Settings → Security → Virus Protection
  6. Write up: What was your biggest security gap? What did you fix?

Verification Checklist:

  • ✓ I checked for password reuse
  • ✓ I checked haveibeenpwned
  • ✓ I enabled 2FA on at least one account
  • ✓ I identified a phishing red flag
  • ✓ I confirmed antivirus is active

Job Connection:

Security analysts run this exact checklist on employee accounts. "User awareness training" and "credential hygiene checks" are standard IT security tasks in every organization.

PROJECT 5 — Capstone

IT Incident Report

+100 XP

~30 min

Write a professional IT incident report for a real or simulated problem you've solved. This is the capstone project that proves you're ready for Level 1.

▸ Show Instructions

Choose ONE of these scenarios (or use a real problem you solved):

  • Scenario A: Computer was running slow — you diagnosed and fixed it
  • Scenario B: Website wouldn't load — you diagnosed the network issue
  • Scenario C: You helped someone avoid a phishing attack
  • Real experience: Any IT problem you've actually solved

Write your report with these sections:

  1. Date/Time: When did this happen?
  2. Problem Summary: What was wrong? What were the symptoms?
  3. Diagnosis: What tools/commands did you use? What did you find?
  4. Root Cause: What was the actual reason for the problem?
  5. Resolution: Exactly what steps did you take to fix it?
  6. Prevention: What can be done to prevent this in the future?

Verification — Your report should:

  • ✓ Have all 6 sections filled out
  • ✓ Use technical terms from the Level 0 glossary
  • ✓ Include specific commands or tools used
  • ✓ Explain WHY the fix worked (not just what you did)
  • ✓ Be something you could share with a hiring manager

Job Connection:

Every IT organization requires incident reports. This format is used by help desks, NOCs, SOCs, and DevOps teams worldwide. A portfolio of incident reports is powerful evidence of IT capability in a job interview.
